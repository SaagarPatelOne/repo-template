# Security Policy

## Reporting a vulnerability

Do not report suspected vulnerabilities in a public issue.

Preferred path:

- Use GitHub private vulnerability reporting if it is enabled for the repository.

Fallback path:

- Email `saagarpatel08@gmail.com` with the subject line `Security report`.
- Include the affected repository, impact, steps to reproduce, and any evidence needed to validate the report.

## Scope

This is the default security guidance for repositories created from this template. A repository may replace it with a more specific policy later.
